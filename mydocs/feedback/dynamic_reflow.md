2026년의 개발 환경은 확실히 과거와 다릅니다. AI와의 페어 프로그래밍을 통해 복잡한 알고리즘 구현의 진입장벽이 낮아진 만큼, 이제는 **'어떻게 구현하느냐'보다 '어떤 최상위 아키텍처를 설계하느냐'**가 소프트웨어의 수명을 결정합니다.

기술적 타협 없이, Rust의 성능을 극한으로 끌어올리면서도 현대적인 워드프로세서가 갖춰야 할 **최고 수준의 구현 전략** 3가지를 제안합니다.

---

## 1. 텍스트 레이아웃: 최적화된 Knuth-Plass 알고리즘

단순히 글자를 채우는 것이 아니라, 문단 전체의 '시각적 농도'를 계산하여 최적의 줄바꿈 위치를 찾는 방식입니다.

* **구현 원리:** 각 줄바꿈 가능 지점을 노드로, 줄의 '불균형도(Demerits)'를 가중치로 하는 유향 비순환 그래프(DAG)를 생성합니다. 이후 **다익스트라(Dijkstra) 알고리즘**을 통해 전체 문단의 페널티 합이 최소가 되는 최단 경로를 찾습니다.
* **페널티 수식 (D):** 
$$D = (a + b \cdot \text{badness})^2 + \text{penalties}$$

여기서 `$\text{badness}$`는 글자 간격이 표준에서 얼마나 벗어났는지를 나타내며, Rust에서는 이를 SIMD를 활용해 문단 전체를 병렬로 계산하여 0.1ms 단위의 성능을 낼 수 있습니다.
* **HWP 특화:** 한글의 자조 결합 특성과 CJK 줄바꿈 규칙(Kinsoku Shori)을 페널티 로직에 포함하여, 한컴오피스 이상의 미려한 레이아웃을 보장합니다.

---

## 2. 상태 관리: 영속적 데이터 구조(Persistent Data Structures)

완벽한 Undo/Redo와 메모리 안정성을 위해 **불변성(Immutability)**과 **구조적 공유(Structural Sharing)**를 채택합니다.

* **아키텍처:** 문서의 전체 트리를 직접 수정하는 대신, 변경된 부분만 새 노드로 생성하고 나머지는 기존 메모리 주소를 공유하는 **B-Tree** 또는 **RRB-Tree** 기반의 데이터 구조를 사용합니다.
* **장점:**
* **Undo/Redo:** 이전 상태의 Root 포인터만 저장하면 되므로 즉각적이고 메모리 효율적인 무한 Undo가 가능합니다.
* **Concurrency:** Rust의 `Arc`를 활용하면 백그라운드 스레드에서 레이아웃을 계산하는 동안 메인 스레드에서 사용자가 텍스트를 입력해도 데이터 경합(Data Race)이 원천 차단됩니다.


* **AI 시너지:** 상태 변화가 명확한 함수형 패러다임을 따르므로, AI가 버그를 추론하거나 기능을 확장하기에 매우 유리한 구조가 됩니다.

---

## 3. 인터랙션: 가상 입력(Virtual Input) & IME 레이어

운영체제(OS)의 IME와 통신하는 것은 가장 까다로운 부분 중 하나입니다.

* **구현 전략:** OS의 네이티브 IME 이벤트를 가로채어, 아직 확정되지 않은 '조합 중인 글자(Composition String)'를 **임시 엔티티(Temporary Entity)**로 렌더링합니다.
* **정교한 커서 제어:** * 커서 위치는 렌더링된 글리프(Glyph)의 실제 너비를  rust wasm 으로부터 피드백 받아 계산합니다.
* 사용자가 입력 중인 글자가 페이지 경계에 걸칠 경우, 레이아웃 엔진이 즉시 개입하여 해당 글자를 다음 줄이나 다음 페이지로 넘기는 **실시간 가변 플로우**를 구현합니다.



---

## 4. 통합 아키텍처: ECS (Entity Component System)

워드프로세서의 각 요소를 게임 엔진처럼 관리합니다.

| 레이어 | 담당 역할 | 기술 스택 |
| --- | --- | --- |
| **Data Layer** | 문서 구조, 불변 트리 상태 관리 | Rust Immutable Collections |
| **Logic Layer** | 레이아웃 계산, Knuth-Plass, 가변 플로우 | Rust Rayon (Parallelism) |
| **Render Layer** | 벡터 그래픽 출력, 폰트 셰이핑 | ThorVG + rustybuzz |
| **Platform Layer** | OS 이벤트, IME, 파일 I/O | SDL2 / WASM |
