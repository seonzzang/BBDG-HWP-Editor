# Task 231: 누름틀 편집 UI

## 목표

웹 편집기에서 누름틀(ClickHere) 필드를 클릭하여 내용을 입력/수정할 수 있다.

## 현재 상태 (Task 229-230 완료 후)

- ClickHere 필드 파싱 및 렌더링 완료
- 빈 필드: 안내문 빨간 기울임체 표시
- WASM API: getFieldList/getFieldValue/setFieldValue 구현 완료
- 문제: 클릭 시 커서가 필드 영역에 진입하지 않음 (안내문 텍스트는 char_start: None)

## 구현 계획

### 1단계: 필드 클릭 진입 + 커서 배치

hitTest가 필드 정보를 반환하도록 확장하고, 클릭 시 필드 편집 모드 진입.

**Rust 측**:
- hitTest 결과에 필드 정보 추가 (isField, fieldId, fieldType)
- 빈 필드 클릭 시에도 커서 위치 반환 (안내문 영역 → 필드 시작 위치로 매핑)

**프론트엔드**:
- `input-handler-mouse.ts`: 필드 클릭 감지 → 커서를 필드 시작 위치로 이동
- `main.ts`: 상태 표시줄에 필드 이름 표시

### 2단계: 필드 내 텍스트 입력/삭제

필드 영역 내 텍스트 편집 동작 구현.

**Rust 측**:
- 필드 내 텍스트 삽입 시 field_range 자동 확장
- 필드 내 텍스트 삭제 시 field_range 자동 축소
- 안내문 ↔ 사용자 입력 전환 (빈→입력: 안내문 숨기고 입력값 표시)

**프론트엔드**:
- 입력 시 기존 insertText 흐름 활용 (필드 범위 내 위치 유지)
- 삭제 시 필드 시작 이전으로 커서 이동 방지

### 3단계: F11 블록 선택 + 상태 표시줄

**프론트엔드**:
- F11 키: 현재 필드 내 텍스트 전체 블록 선택
- 상태 표시줄: 필드 이름/안내문 표시

## 산출물

- 누름틀 클릭 → 커서 진입 → 텍스트 입력/삭제
- F11 블록 선택
- 상태 표시줄 필드 정보 표시
