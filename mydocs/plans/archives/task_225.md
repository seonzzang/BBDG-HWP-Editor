# Task 225: synam-001.hwp 페이지 하단 빈 Return 문단 overflow 조사

## 문제 현상

synam-001.hwp에서 페이지 하단의 빈 `<Return>` 문단들이 다음 페이지로 넘어가는 overflow가 발생한다.
한컴에서는 같은 페이지에 유지되지만, 우리 렌더러에서는 정직하게 높이를 계산하여 다음 페이지로 넘김.

## 원인 분석

HWP 스펙 **표 132: 구역 정의 속성** bit 19: **"빈 줄 감춤 여부"**

- synam-001.hwp의 `section_def.flags = 0x00080000` → bit 19 = 1 (빈 줄 감춤 활성)
- 새로 만든 일반 문서에서는 이 플래그가 꺼져 있어 정상 동작
- 현재 파서에서 이 플래그를 읽지 않고, 페이지네이션에서도 반영하지 않음

## 조사 필요 사항

1. "빈 줄 감춤"의 정확한 동작 범위:
   - 페이지 하단에서만 작동하는지, 문서 전체에서 빈 줄 높이를 0으로 처리하는지
   - "빈 줄"의 정의: 텍스트가 완전히 비어있는 문단만 해당하는지, 공백만 있는 문단도 포함하는지
   - 연속 빈 줄 중 몇 개까지 감추는지 (1개? 전부?)

2. 한컴 도움말에서 관련 설명 확인

## 수행 계획

### 1단계: 조사 및 동작 정의
- 한컴 도움말에서 "빈 줄 감춤" 관련 설명 확인
- 파서에서 bit 19 읽어서 `SectionDef`에 `hide_empty_line: bool` 필드 추가

### 2단계: 페이지네이션 반영
- 빈 줄 감춤 플래그가 활성일 때, 페이지 하단 빈 문단의 높이를 0으로 처리
- 정확한 동작 범위에 따라 구현 (페이지 하단만/전체)

### 3단계: 테스트 및 검증
- synam-001.hwp 기존 overflow 해소 확인
- 다른 샘플 문서에 영향 없는지 확인

## 결과

- 41페이지 → 35페이지로 변경, 한컴과 동일
- 694개 테스트 통과, WASM 빌드 완료
- 구현: 페이지 끝에서 빈 문단이 overflow 유발 시 최대 2개/페이지 높이 0 처리
- 레이아웃에서 hidden 문단은 렌더링(문단부호 표시)하되 y_offset 변경 없음
