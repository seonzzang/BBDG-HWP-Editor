# 타스크 32: 서식 툴바 구현 — 수행계획서

## 1. 목표

webhwp과 동일한 서식 툴바를 rhwp 편집 모드에 추가한다.

### 기능 요구사항

| 기능 | 설명 |
|------|------|
| 글꼴 이름 | 현재 캐럿 위치의 글꼴 표시, 드롭다운으로 변경 |
| 글꼴 크기 | pt 단위 표시, 드롭다운/증감 버튼으로 변경 |
| 굵게/기울임/밑줄/취소선 | 토글 버튼, 현재 상태 반영 |
| 글자 색 | 색상 표시기 + 팔레트 팝업 |
| 형광펜 | 배경색 표시기 + 팔레트 팝업 |
| 문단 정렬 | 왼쪽/가운데/오른쪽/양쪽/배분 5종 |
| 줄 간격 | 현재 값 표시 + 팝업 |
| 글머리/번호 | 토글 버튼 |
| 들여쓰기/내어쓰기 | 버튼 |
| 실행취소/다시실행 | 비활성 표시 (별도 타스크) |

### 속성 반영 요구사항

- 캐럿 이동 시 해당 위치의 글자/문단 속성을 툴바에 실시간 반영
- 텍스트 선택 시 선택 범위의 공통 속성 반영 (혼합 시 indeterminate)
- 표 셀 내부에서도 동일하게 동작

## 2. 현재 상태

### 이미 구현됨
- 텍스트 레이아웃 JSON에 fontFamily, fontSize, bold, italic 포함
- 캐럿/선택 시스템 (text_selection.js)
- WASM 텍스트 편집 API (insertText, deleteText 등)

### 구현 필요
- 텍스트 레이아웃 JSON에 underline, strikethrough, textColor, alignment 추가
- CharShape/ParaShape 변경 Rust 로직
- WASM 서식 적용 API
- 서식 툴바 UI (HTML/CSS/JS)

## 3. 구현 범위

### 포함
- 서식 툴바 UI 전체
- 속성 조회 및 실시간 반영
- 글자 서식 적용 (굵게, 기울임, 밑줄, 취소선, 글꼴, 크기, 색상)
- 문단 서식 적용 (정렬, 줄 간격, 들여쓰기)

### 제외 (별도 타스크)
- Undo/Redo 시스템
- 글머리 기호/번호 매기기 (UI만 표시, 기능은 미구현)
- 눈금자

## 4. 기술적 위험

| 위험 | 대응 |
|------|------|
| CharShapeRef 분할 복잡도 | 단위 테스트로 엣지 케이스 검증 |
| 서식 적용 후 재조판 성능 | 변경된 섹션만 재조판하도록 최적화 |
| HWP 저장 시 새 CharShape 직렬화 | 기존 serializer가 doc_info 전체 재빌드 지원 |

## 5. 일정

6단계 구현, 각 단계별 완료보고서 작성 후 승인 요청
