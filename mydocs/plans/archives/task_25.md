# 타스크 25: 컨트롤 객체 선택 및 셀 선택 — 수행계획서

## 목표

표/이미지/도형 등 컨트롤 객체를 선택하는 메커니즘과, 표의 셀을 단위로 선택하는 기능을 구현한다. 이 기능은 향후 표 구조 편집(B-904~B-906)의 전제 조건이다.

## 현재 상태 분석

### 편집 모드 현황
- **텍스트 편집 모드**: 본문/셀 내부에서 캐럿 + 텍스트 입력/삭제 동작 (완료)
- **객체 선택 모드**: 미구현 — 표/이미지/도형을 클릭해도 내부 텍스트 편집으로만 진입
- **셀 선택 모드**: 미구현 — 셀 단위 선택/범위 선택 불가

### 현재 아키텍처
```
마우스 클릭 → hitTest(x,y) → TextRun 매칭 → 캐럿 설정 → 텍스트 편집
```

### 목표 아키텍처
```
마우스 클릭 → hitTest(x,y) 확장
  ├── TextRun 내부 → 텍스트 편집 모드 (기존)
  ├── 표 테두리/여백 영역 → 표 객체 선택 모드 (신규)
  ├── 이미지/도형 영역 → 객체 선택 모드 (신규)
  └── 셀 선택 모드 진입 (F5 또는 Esc) → 셀 범위 선택 (신규)
```

## 구현 범위

### B-901: 컨트롤 객체 선택
1. **렌더트리에서 컨트롤 영역 정보 제공** — 표/이미지/도형의 바운딩 박스를 JS에 전달
2. **히트테스트 확장** — 클릭 좌표가 컨트롤 영역에 해당하면 객체 선택 반환
3. **선택 시각화** — 선택된 객체 주위에 테두리 + 리사이즈 핸들 표시
4. **키보드 연동** — Esc로 선택 해제, Delete로 객체 삭제 (표 제외)

### B-902: 셀 선택 모드
1. **셀 영역 정보 제공** — 각 셀의 바운딩 박스를 JS에 전달
2. **셀 선택 진입** — 표 객체 선택 상태에서 Enter 또는 F5로 셀 선택 모드 진입
3. **셀 범위 선택** — 클릭+드래그 또는 Shift+방향키로 셀 범위 선택
4. **셀 선택 시각화** — 선택된 셀에 배경 하이라이트
5. **모드 전환** — Enter로 셀 내부 텍스트 편집 진입, Esc로 표 객체 선택으로 복귀

## 편집 모드 상태 전이

```
[텍스트 편집] ←Enter→ [셀 선택] ←Enter/Esc→ [표 객체 선택] ←Esc→ [선택 없음]
      ↑                                              ↑
      └── 셀 내부 클릭                      표 테두리 클릭
```

## 변경 대상 파일

| 파일 | 변경 내용 |
|------|-----------|
| `src/wasm_api.rs` | getPageControlLayout API (컨트롤/셀 바운딩 박스) |
| `web/text_selection.js` | 히트테스트 확장, 모드 상태 관리 |
| `web/editor.js` | 객체 선택/셀 선택 키보드 핸들러, 시각화 |

## 제외 범위
- 객체 리사이즈 (드래그로 크기 변경)
- 객체 이동 (드래그로 위치 변경)
- 셀 병합/분할, 행/열 추가/삭제 (B-904, B-905)
- 표/셀 속성 대화상자 (B-906)
