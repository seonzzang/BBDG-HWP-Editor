# 타스크 197 수행계획서 — 다양한 줄간격 문서의 페이지 넘김 검증

## 목적

문단별로 다양한 줄간격(100%, 160%, 250%, 300%, 고정값 등)이 적용된 문서에서 페이지 넘김 계산이 정확한지 검증한다.

## 배경

- 줄간격 타입: Percent, Fixed, SpaceOnly, Minimum (4종)
- 줄간격은 `LineSeg.line_spacing` 값으로 반영되며, 페이지네이션 시 `line_height + line_spacing`이 누적되어 페이지 넘침 여부를 결정
- 타스크 196에서 `vertical_pos` 누적 계산 버그를 수정했으므로, 다양한 줄간격에서도 정상 동작하는지 확인 필요

## 검증 방법

1. **네이티브 단위 테스트**: 다양한 줄간격 조합의 문단으로 페이지 넘김 위치가 정확한지 Rust 테스트로 검증
2. **E2E 브라우저 테스트**: WASM `applyParaFormat` API로 줄간격을 변경하고 스크린샷으로 시각 확인

## 검증 시나리오

| 시나리오 | 줄간격 설정 | 기대 결과 |
|----------|------------|-----------|
| S1 | 모든 문단 160% (기본) | 약 35줄에서 페이지 넘김 |
| S2 | 모든 문단 100% (빽빽) | 더 많은 줄 수용 후 넘김 |
| S3 | 모든 문단 300% (넓음) | 더 적은 줄에서 넘김 |
| S4 | 혼합: 160% → 300% → 100% | 줄간격 변경 시점에서 올바른 높이 누적 |
| S5 | 고정 줄간격 (Fixed) | 폰트 크기와 무관하게 고정 피치 |

## 변경 범위

- `src/document_core/commands/text_editing.rs` — 줄간격 페이지 넘김 단위 테스트 추가
- `rhwp-studio/e2e/line-spacing.test.mjs` — E2E 줄간격 검증 테스트 (신규)
