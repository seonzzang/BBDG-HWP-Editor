# Task 213 수행계획서

## 제목
kps-ai.hwp p66 표 분할 시 빈 페이지 생성 및 중첩 표 잔여 콘텐츠 누락 수정 (B-014)

## 현상
- **파일**: kps-ai.hwp, 66~68페이지
- **문제 1**: p66의 표(para 674, 3행)가 분할될 때 row 0~1(51.7px)만 한 페이지에 배치하고 row 2(946px)를 다음 페이지로 통째로 넘김 → p66이 거의 빈 페이지 → **수정 완료**
- **문제 2**: p66에서 row 0~2를 split_end_limit=862.88로 배치 후, row 2의 나머지 콘텐츠(~83px)가 다음 페이지에 이어지지 않음 → remaining_content_for_row가 0 반환 → **수정 완료**
- **문제 3 (현재)**: p67~p68이 빈 문단(para 675~751, text="", controls=0)으로만 구성되어 빈 페이지로 렌더링됨. 이 빈 문단들은 표 셀 내부 문단이 구역 수준으로 잘못 pagination됨

## 원인 분석

### 문제 3의 핵심
para 674의 표(row 2)에는 중첩 표가 포함된 셀이 있음. 이 셀의 중첩 표 내부에는 다수의 문단(para 675~751)이 존재. 이 문단들이 파서/모델 수준에서는 셀 내부에 올바르게 소속되어 있으나, pagination engine이 이들을 **외부 문단**으로 취급하여 별도 페이지에 배치.

가능한 원인:
1. pagination engine의 `paginate_paragraphs()`가 셀 내부 문단을 구역 수준 문단 목록에서 순회하면서 표 범위를 넘어서 계속 진행
2. 표 컨트롤의 셀 문단 인덱스가 구역 문단 인덱스와 혼동되어 para 675~를 외부로 판단

## 수정 방안
1. pagination engine에서 표 다음의 문단 인덱스가 셀 내부 문단인지 확인하는 로직 조사
2. 빈 문단(text="", controls=0)이 연속으로 나오는 경우 pagination에서 제외하거나 올바른 표 셀에 소속시킴
3. 기존 테스트 회귀 확인
