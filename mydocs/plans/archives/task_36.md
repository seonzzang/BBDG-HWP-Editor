# 타스크 36: 표 테두리 처리 고도화 - 수행계획서

## 1. 현재 상태 분석

### 구현 완료 항목
- BorderFill 파싱 (인터리브 형식, 4방향 + 대각선)
- 그라데이션 채우기 파싱 (`parse_fill` → `GradientFill`)
- 셀 테두리 SVG 렌더링 (단일선, 이중선, 삼중선)
- 테두리 굵기 16단계 매핑 (0.1mm ~ 5.0mm)
- 선 종류별 대시 패턴 (Solid, Dash, Dot, DashDot, DashDotDot)

### 발견된 문제점

#### P0: 그라데이션 채우기 미렌더링 (k-water-rfp.hwp 1페이지)
- **현상**: 1페이지 표의 상단/하단 장식 셀이 투명하게 렌더링됨
- **원인**: `ResolvedBorderStyle`이 `fill_color: Option<ColorRef>`(단색)만 지원
- `resolve_single_border_style()`에서 `FillType::Gradient`를 완전히 무시 (`_ => None`)
- 표 구조: 3행×1열, Row 0/2 (높이 2.9mm)에 그라데이션 채우기 (연파란→진파란)
- Row 0/2의 border_fill_id=18 → BF[17]: borders=None, fill=Gradient
- **영향**: 장식적 셀 배경이 투명하게 되어 표의 시각적 프레임이 사라짐

#### P1: 인접 셀 테두리 중복 그리기 (Overdrawing)
- 각 셀이 독립적으로 4방향 테두리를 모두 그림
- 인접한 두 셀의 경계에서 테두리가 2번 그려짐
- 반투명 색상이나 다른 굵기의 테두리가 겹치면 시각적 결함 발생

#### P2: 테두리 꼭짓점 (Corner) 처리 미비
- 수직선과 수평선이 교차하는 꼭짓점에서 선 끝이 깔끔하지 않음
- 굵은 테두리에서 특히 눈에 띔

#### P3: 페이지 분할 시 테두리 처리 미완성
- `layout_partial_table`에서 분할된 표의 상단/하단 경계선 처리 부재
- `table.border_fill_id`를 활용한 경계선 스타일 미적용

#### P4: 대각선 테두리 렌더링 미구현
- `DiagonalLine` 구조체는 파싱되지만 렌더링에서 미사용

#### P5: Wave/3D 선 종류 미지원
- Wave(12), DoubleWave(13), 3D(14~17) 타입이 Solid로 대체됨

## 2. 목표

표 테두리 및 채우기 렌더링 품질을 한/글 원본과 가깝게 개선한다.

| 순위 | 항목 | 영향도 | 난이도 |
|------|------|--------|--------|
| 1 | **그라데이션 채우기 렌더링** | 높음 | 중 |
| 2 | 인접 셀 테두리 중복 제거 | 높음 | 중 |
| 3 | 테두리 꼭짓점 처리 | 중 | 중 |
| 4 | 페이지 분할 시 경계선 처리 | 중 | 중 |

## 4. 접근 방법

### 4.1 그라데이션 채우기 렌더링

1. `ResolvedBorderStyle`에 `gradient: Option<GradientFill>` 필드 추가
2. `resolve_single_border_style`에서 그라데이션 데이터 전달
3. 레이아웃 엔진에서 그라데이션 → SVG `<linearGradient>` / `<radialGradient>` 변환
4. `<rect>` 요소에 gradient fill 적용

### 4.2 엣지 기반 테두리 렌더링

현재는 각 셀이 자신의 4방향 테두리를 모두 그리는 방식이다.
이를 **엣지(edge) 단위**로 전환하여 인접 셀 간 공유 테두리를 한 번만 그리도록 한다.

- 표 전체의 수평/수직 엣지 목록을 생성
- 각 엣지에 대해 인접한 두 셀의 테두리 스타일을 비교
- 우선순위 규칙에 따라 하나를 선택하여 렌더링

## 5. WebHWP 참고사항

- WebHWP는 **Canvas 2D API**로 표 테두리를 렌더링 (CSS가 아님)
- `webhwp.css`에는 표 관련 스타일 없음 (canvas, cursor, tooltip만)
- 핵심 함수: `bbr()` (기본선), `Cbr()` (코너), `Tbr()` (디스패처)
- 인접 셀 정보를 코너 처리에 활용
- 패턴 채우기 8종, 색상 변환 `CREFtoRGB()` 지원
