# 타스크 37: 클립보드 복사 및 붙여넣기 기능 구현 - 수행계획서

## 배경

백로그 B-305 "복사/붙여넣기 (클립보드 연동)" 항목이다.
현재 에디터에는 Ctrl+C로 선택된 텍스트를 플레인 텍스트로 클립보드에 복사하는 기능만 존재한다 (`text_selection.js` line 755-768).

## 목표

1. **HWP 프로그램 → 우리 에디터**: HWP 편집기에서 컨트롤 객체(표, 이미지, 도형 등)를 복사한 후 우리 에디터에 붙여넣기
2. **우리 에디터 → HWP 프로그램**: 우리 에디터에서 선택한 내용을 복사하여 HWP 편집기에 붙여넣기
3. **한컴 OLE 클립보드 포맷 지원**

### 한컴 클립보드 포맷 분석 (참조: tech.hancom.com)

HWP 편집기는 복사 시 다음 포맷을 클립보드에 등록한다:
- **Native (OLE 1.0)**: HWP 바이너리 데이터
- **Data Object / Embedded Object (OLE 2.0)**: OLE 2.0 포맷
- **HTML**: HTML 마크업 (텍스트 + 서식)
- **Text**: 플레인 텍스트

붙여넣기 우선순위: Native → HTML → Text

### 브라우저 클립보드 API 제약

웹 브라우저에서 사용 가능한 클립보드 포맷:
- `text/plain` - 플레인 텍스트 (읽기/쓰기)
- `text/html` - HTML (읽기/쓰기)
- `image/png` - PNG 이미지 (읽기/쓰기)
- OLE/Native 포맷 - **직접 접근 불가**

**핵심 제약**: 브라우저 Clipboard API는 OLE/Native 바이너리 포맷에 직접 접근할 수 없다. 따라서:
- **HWP → 우리 에디터**: HWP가 클립보드에 등록하는 HTML 포맷을 파싱하여 활용
- **우리 에디터 → HWP**: HTML 포맷으로 클립보드에 등록하여 HWP의 HTML 붙여넣기 경로 활용
- **우리 에디터 ↔ 우리 에디터**: HWP 바이너리를 base64로 인코딩하여 HTML data attribute에 포함하거나, 내부 메모리 버퍼 활용

## 구현 전략

### 복사 (우리 에디터 → 외부)

1. 선택 영역의 단락/컨트롤을 추출하여 미니 Document IR 구성
2. Document IR → HWP 바이너리 직렬화 (기존 `serialize_hwp()` 활용)
3. 동시에 HTML 표현 생성 (서식 보존)
4. 클립보드에 `text/html` + `text/plain` 등록

### 붙여넣기 (외부 → 우리 에디터)

1. 클립보드에서 데이터 읽기 (`navigator.clipboard.read()`)
2. 포맷 우선순위: 내부 HWP 바이너리 > HTML > 플레인 텍스트
3. HTML 파싱 시 표/이미지 구조 복원
4. Document IR에 삽입

### 내부 복사/붙여넣기 (우리 에디터 ↔ 우리 에디터)

1. 복사 시 WASM 측에서 선택 영역을 HWP 바이너리로 직렬화하여 메모리에 보관
2. 붙여넣기 시 메모리의 HWP 바이너리를 파싱하여 삽입
3. 서식, 표 구조, 이미지 등 완벽 보존

## 범위

### 포함

- 텍스트 복사/붙여넣기 (서식 포함)
- 표 복사/붙여넣기 (구조 + 내용)
- 이미지 복사/붙여넣기
- 다중 단락 복사/붙여넣기
- HWP ↔ 우리 에디터 간 HTML 포맷 교환
- 내부 클립보드 (HWP 바이너리 기반)

### 제외 (후속 타스크)

- 수식, 차트 등 미구현 컨트롤의 복사
- 다른 오피스 프로그램(MS Word, Google Docs) 간 교환
- 드래그 앤 드롭

## 위험 요소

| 위험 | 대응 |
|------|------|
| 브라우저 OLE 직접 접근 불가 | HTML 포맷을 중간 매개체로 활용 |
| HWP HTML 클립보드의 구조/스타일 불완전 | 파싱 시 fallback 처리, 점진적 개선 |
| 대용량 표/이미지 복사 시 성능 | base64 이미지 크기 제한, 비동기 처리 |
| 크로스 브라우저 Clipboard API 차이 | Feature detection, fallback (execCommand) |

- HWP 편집기 → 우리 에디터 텍스트 붙여넣기 (HTML 경유)
- 우리 에디터 → HWP 편집기 텍스트 붙여넣기 (HTML 경유)
