# 타스크 198 수행계획서 — 표 페이지 경계 분할 처리 검증

## 목적

표가 페이지 경계에 걸칠 때 행 단위로 정상 분리되는지 검증하고, 관련 버그를 탐색·수정한다.

## 배경

- 표 분할은 `split_table_rows()` (engine.rs)에서 행 단위로 처리
- `MeasuredTable`의 `cumulative_heights`를 이진 탐색하여 분할 위치 결정
- 행 내 분할(intra-row split)도 지원: 셀 내용이 길 경우 행 중간에서 분리
- 기존 백로그 B-011: hwpp-001.hwp page 23 표 overflow 버그 존재

## 검증 방법

1. **기존 버그 재현**: B-011 (hwpp-001.hwp page 23) 표 overflow 원인 분석
2. **네이티브 단위 테스트**: 다양한 표 크기/행 수로 페이지 분할 검증
3. **E2E 브라우저 테스트**: 웹에서 표 생성 후 페이지 경계 분할 시각 검증
4. **SVG 출력 비교**: 분할된 표의 렌더링 정확성 확인

## 검증 시나리오

| 시나리오 | 설명 | 기대 결과 |
|----------|------|-----------|
| S1 | 10행 표가 페이지 하단 1/3에서 시작 | 일부 행은 1페이지, 나머지는 2페이지에 분리 |
| S2 | 50행 대형 표 | 여러 페이지에 걸쳐 행 단위 분리 |
| S3 | 제목행 반복 설정 | 2페이지부터 제목행 반복 표시 |
| S4 | 셀 내용이 매우 긴 행 | 행 내 분할(intra-row split) |
| S5 | B-011 재현 | body area 초과 버그 원인 파악 |
