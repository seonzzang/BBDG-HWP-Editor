# 타스크 196 구현 계획서 — 웹편집기 텍스트 플로우 처리

## 1단계: E2E 테스트 인프라 구축 + 기본 동작 검증

### 목표
- Puppeteer 기반 E2E 테스트 프레임워크 구성
- WASM 빌드 → Vite dev server → headless Chrome 자동화 파이프라인 구축
- 빈 문서 로드 및 기본 텍스트 입력 동작 검증

### 구현
- `rhwp-studio/e2e/` 디렉토리 생성
- `e2e/helpers.ts`: Puppeteer 연결, 페이지 로드, 편집 영역 포커스 유틸리티
- `e2e/text-flow.test.ts`: 기본 테스트 케이스
  - 빈 문서 로드 후 편집 영역 클릭
  - 텍스트 입력 → 화면에 렌더링 확인
- `package.json`에 `"e2e"` 스크립트 추가

### 검증
- 헤드리스 Chrome에서 텍스트 입력이 캔버스에 반영되는지 확인
- 스크린샷 캡처로 시각적 확인

## 2단계: 줄바꿈 및 줄간격 검증

### 목표
- 긴 텍스트 입력 시 자동 줄바꿈 동작 확인
- 줄간격이 올바르게 적용되는지 검증

### 테스트 시나리오
- 한 줄을 넘는 긴 텍스트 입력 → 자동 줄바꿈 발생 확인
- 줄바꿈 후 텍스트가 두 줄로 렌더링되는지 확인 (WASM API로 line count 확인)
- 줄간격 값이 레이아웃에 반영되는지 확인

### 검증
- cargo test로 Rust 측 줄바꿈 로직 정상 확인
- E2E 스크린샷으로 시각적 줄바꿈 확인

## 3단계: 엔터(문단 분리) 및 Backspace(문단 병합) 검증

### 목표
- Enter 키로 문단 분리가 올바르게 동작하는지 확인
- Backspace로 문단 시작에서 이전 문단과 병합되는지 확인

### 테스트 시나리오
- 텍스트 입력 → Enter → 추가 텍스트 → 문단이 2개로 분리 확인
- 두 번째 문단 시작에서 Backspace → 문단 병합 확인
- 커서 위치가 올바르게 갱신되는지 확인

### 검증
- WASM `getParagraphCount()` 로 문단 수 변화 확인
- E2E 스크린샷으로 시각적 확인

## 4단계: 페이지 넘김 검증 및 버그 수정

### 목표
- 텍스트를 충분히 입력하여 1페이지를 넘길 때 2페이지 생성 확인
- 페이지 수 변화에 따른 스크롤 영역 갱신 확인
- 발견된 버그 수정

### 테스트 시나리오
- Enter 키를 반복 입력하여 페이지 오버플로우 유발
- 2페이지 생성 확인 (`wasm.pageCount` 변화)
- 스크롤하여 2페이지 내용 확인
- Backspace로 1페이지로 복귀 확인

### 검증
- 페이지 수 동적 증감 확인
- 스크린샷으로 페이지 넘김 시각적 확인
- 발견된 모든 버그 수정 후 회귀 테스트
