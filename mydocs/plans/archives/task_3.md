# 타스크 3 - 수행 계획서: HWP 파서 구현

## 목표

HWP 5.0 파일의 바이너리 데이터를 파싱하여 타스크 2에서 설계한 IR(Document Model)로 변환하는 파서를 구현한다. 파서 완성 후 `HwpDocument::from_bytes()` API를 통해 실제 HWP 파일을 로드할 수 있어야 한다.

## 현황 분석

### 이미 완성된 부분

| 모듈 | 상태 | 비고 |
|------|------|------|
| IR 데이터 모델 (src/model/) | 완성 | 12개 파일, 31개 테스트 |
| 렌더링 엔진 (src/renderer/) | 완성 | 8개 파일, 44개 테스트 |
| WASM API (src/wasm_api.rs) | 완성 | 12개 테스트 |
| 외부 크레이트 (cfb, flate2, byteorder) | 준비됨 | Cargo.toml 등록 완료 |

### 구현 필요 부분

| 구현 항목 | 설명 |
|----------|------|
| CFB 컨테이너 파싱 | OLE 구조에서 스트림 추출 (cfb 크레이트 활용) |
| 스트림 압축 해제 | FLATE 압축된 스트림 디코딩 (flate2 크레이트 활용) |
| 파일 헤더 파싱 | 버전, 플래그(압축/암호화/배포) 파싱 |
| DocInfo 파싱 | ID 매핑 (폰트, 글자모양, 문단모양, 테두리, 스타일) |
| BodyText 파싱 | 섹션/문단/컨트롤/표/도형 레코드 파싱 |
| BinData 추출 | 임베디드 이미지/OLE 바이너리 데이터 추출 |

## HWP 5.0 파일 구조 개요

```
HWP 파일 (OLE/CFB 컨테이너)
├── FileHeader          (256 bytes, 비압축)
├── DocInfo             (레코드 스트림, 압축 가능)
├── BodyText/
│   ├── Section0        (레코드 스트림, 압축 가능)
│   ├── Section1
│   └── ...
├── BinData/
│   ├── BIN0001.png
│   └── ...
├── PrvText             (미리보기 텍스트)
├── PrvImage            (미리보기 이미지)
└── DocOptions/         (기타 옵션)
```

### 레코드 구조

```
┌──────────────────────────────────────┐
│ 레코드 헤더 (4 bytes)                 │
│  bits 0~9:   태그 ID (0~1023)        │
│  bits 10~19: 레벨 (0~1023)           │
│  bits 20~31: 크기 (0~4095)           │
│  크기 == 4095이면 → 추가 4바이트 크기  │
├──────────────────────────────────────┤
│ 레코드 데이터 (가변 길이)              │
└──────────────────────────────────────┘
```

## 구현 범위

### 1차 지원 (이번 타스크)

- FileHeader 파싱 (버전, 플래그)
- DocInfo: 폰트, 글자모양, 문단모양, 테두리채우기, 스타일
- BodyText: 문단(텍스트, 글자모양참조, 줄세그먼트), 구역정의, 단정의
- 컨트롤: 표, 직사각형, 직선, 타원, 그림
- BinData: 임베디드 이미지 추출
- 머리말/꼬리말

### 미지원 (향후)

- 암호화된 문서
- 배포용 문서
- 수식, 차트, OLE 개체
- 각주/미주 (구조는 정의됨)
- 매크로, 양식

## 기대 결과

- `HwpDocument::from_bytes(&data)`로 실제 HWP 파일 로드 가능
- `rhwp info sample.hwp` CLI 명령으로 문서 정보 출력
- `rhwp export-svg sample.hwp` CLI 명령으로 SVG 내보내기 (기본 텍스트/표/도형)
- HWP 파서 단위 테스트 및 통합 테스트
