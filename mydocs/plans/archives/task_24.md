# 타스크 24: 표 안의 텍스트 입력 처리 - 수행계획서

## 1. 개요

현재 텍스트 입력(B-303)은 본문 문단에서만 동작한다. 표(Table) 셀 내부의 문단에 대해서는 히트테스트, 캐럿 배치, 텍스트 입력/삭제가 지원되지 않는다. 이 타스크에서는 표 셀 내부에서도 본문과 동일한 편집 기능이 동작하도록 처리한다.

## 2. 현재 상태 분석

### 동작하는 기능
- 본문 문단 텍스트 입력/삭제 (`insert_text`, `delete_text`)
- 본문 문단 히트테스트 및 캐럿 위치 지정
- 표 렌더링 (셀 배경, 테두리, 셀 내 텍스트 표시)
- 문단 리플로우 및 분리/병합

### 미지원 기능 (이 타스크의 범위)
- 표 셀 클릭 시 히트테스트 → 셀 내 문단/글자 위치 식별
- 셀 내 캐럿 표시 및 이동
- 셀 내 텍스트 입력/삭제
- 셀 간 Tab 키 이동

## 3. 핵심 과제

### 3.1 텍스트 레이아웃 데이터 확장
- `getPageTextLayout()`이 현재 본문 문단의 run만 반환
- 표 셀 내부 문단의 run도 포함하도록 확장 필요
- 각 run에 셀 식별 정보 추가 (tableIdx, cellIdx 또는 controlIndex)

### 3.2 WASM API 확장
- `insert_text`, `delete_text` 등이 셀 내부 문단을 대상으로 동작하도록 확장
- 문단 경로: section → paragraph(표 컨트롤 소유) → control(Table) → cell → cell paragraph

### 3.3 에디터 JS 연동
- 히트테스트 결과에 셀 정보 포함
- 캐럿 위치를 셀 내부 좌표로 관리
- 키보드 이벤트(입력/삭제/방향키)를 셀 문맥으로 전달

## 4. 영향 범위

| 파일 | 변경 내용 |
|------|-----------|
| `src/wasm_api.rs` | 셀 내부 문단 대상 insert/delete API 추가 또는 확장 |
| `src/renderer/layout.rs` | 셀 내 텍스트 레이아웃 run 데이터 생성 |
| `web/editor.js` | 셀 문맥 키보드 이벤트 디스패치 |
| `web/text_selection.js` | 히트테스트, 캐럿, 선택 영역의 셀 지원 |

## 5. 제외 범위

- 셀 내 문단 분리/병합 (Enter 키로 셀 내 새 문단 생성)
- 셀 크기 자동 조절 (텍스트 증가에 따른 셀 높이 변경)
- 표 구조 편집 (행/열 추가/삭제)
