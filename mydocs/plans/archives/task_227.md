# Task 227: 텍스트 블럭 복사/붙여넣기 시 새 페이지 생성 버그 수정

## 문제 설명

빈 문서에서 '가나다라마바사' 입력 후, 전체 문자열 선택(Shift+End) → 복사(Ctrl+C) → End로 커서 이동(선택 해제) → 붙여넣기(Ctrl+V) 시, 새로운 페이지가 생성되면서 다음 페이지에 붙여넣기되는 버그.

**기대 동작**: 같은 페이지의 같은 문단에 텍스트가 이어 붙여짐
**실제 동작**: 새 페이지가 생성되고 2페이지에 텍스트가 붙여넣기됨

## 원인 분석 방향

1. 복사 시 클립보드 문단에 제어문자(\r, 0x000D)가 포함되는지 확인
2. 붙여넣기 시 단일 문단 판정(`clip_count == 1 && controls.is_empty()`)이 올바른지 확인
3. 문단 분할(`split_at`) 후 잘못된 문단이 생성되어 페이지네이션에 영향을 주는지 확인

## 구현 계획

### 1단계: 버그 재현 및 원인 특정
- Rust 단위 테스트로 시나리오 재현 (빈 문서 → 텍스트 삽입 → 복사 → 붙여넣기)
- `copy_selection_native` 결과의 클립보드 문단 내용 검증
- `paste_internal_native` 동작 추적

### 2단계: 버그 수정
- 원인에 따라 copy 또는 paste 로직 수정
- 단위 테스트로 수정 검증

### 3단계: WASM 빌드 및 통합 테스트
- WASM 빌드
- 브라우저에서 실제 시나리오 재현 확인
