# 타스크 13: 수행계획서 - 서식-문단-번호 모양: 시작번호/문단번호 처리

## 1. 개요

HWP 문서의 "서식 > 문단 > 번호 모양"에 해당하는 문단 번호(Numbering)를 파싱하고 렌더링하는 기능을 구현한다.

## 2. 현재 상태 분석

### 구현된 것
- `AutoNumber` (atno 컨트롤): 그림/표/각주 등 자동 번호 — 이미 구현 완료
- `ParaShape.numbering_id`: 파싱되지만 실제 사용되지 않음
- `HWPTAG_NUMBERING` 태그 상수: 정의만 존재, 파싱 로직 없음
- `NumberFormat` 및 `format_number()`: 번호 형식 변환 함수 존재 (재사용 가능)

### 미구현된 것
- `HWPTAG_NUMBERING` 레코드 파싱 (DocInfo)
- `Numbering` 모델 구조체 (번호 정의 데이터)
- `ParaShape.attr1`의 bit 23~24 (문단 머리 모양 종류) 및 bit 25~27 (문단 수준) 활용
- 문단 번호 렌더링 (번호 문자열 생성 및 텍스트 앞에 삽입)
- 수준별 시작 번호 처리
- 번호 카운터 관리 (같은 수준 연속 시 번호 증가, 수준 변경 시 리셋)

## 3. HWP 스펙 기반 설계

### 문단 번호 동작 방식
1. **DocInfo의 HWPTAG_NUMBERING**: 번호 형식 정의 (수준별 형식 문자열, 시작 번호)
2. **ParaShape.attr1 bit 23~24**: 문단 머리 종류 (0=없음, 1=개요, 2=번호, 3=글머리표)
3. **ParaShape.attr1 bit 25~27**: 문단 수준 (1~7수준)
4. **ParaShape.numbering_id**: 어떤 번호 정의를 사용할지 참조 ID

### 번호 형식 문자열 (스펙 표 40)
- 각 수준(1~7)마다 형식 문자열 존재
- `^n`: 레벨 경로 표시 (예: 1.1.1)
- `^N`: 레벨 경로 + 마지막 마침표 (예: 1.1.1.)

### 문단 머리 정보 (스펙 표 41)
- 속성 (4바이트): 정렬, 너비 따름, 자동 내어 쓰기 등
- 너비 보정값 (2바이트)
- 본문과의 거리 (2바이트)
- 글자 모양 아이디 참조 (4바이트)

## 4. 작업 범위

| 항목 | 설명 |
|------|------|
| 모델 | `Numbering` 구조체, `NumberingHead` 구조체 추가 |
| 파서 | DocInfo에서 HWPTAG_NUMBERING 파싱 |
| 파서 | ParaShape.attr1에서 머리 종류/수준 추출 |
| 렌더링 | 문단별 번호 카운터 관리 및 번호 텍스트 생성 |
| 렌더링 | 번호 문자열을 문단 텍스트 앞에 삽입 |

## 5. 기대 결과

- HWP 문서에서 "1. 항목", "가. 항목", "I. 항목" 등의 문단 번호가 올바르게 렌더링됨
- 수준별 시작 번호가 반영됨
- 같은 수준의 연속 문단은 번호가 자동 증가됨
