# 타스크 194 수행계획서 — 머리말/꼬리말 관리 기능

## 목표

타스크 193에서 구현한 머리말/꼬리말 기본 생성·편집 기능을 확장하여, 한컴 워드프로세서와 동일한 관리 기능을 제공한다.

## 현재 상태 (타스크 193 완료)

| 기능 | 상태 |
|------|------|
| 머리말/꼬리말 생성 (Both/Even/Odd) | 완료 |
| 편집 모드 진입/탈출 | 완료 |
| 텍스트 입력/삭제/IME | 완료 |
| 문단 분할/병합 | 완료 |
| 문맥 도구상자 전환 | 완료 |

## 구현 범위

한컴 도움말(`format/header/`)을 참고하여 다음 기능을 구현한다.

### 1. 머리말/꼬리말 삭제

- **내용 삭제**: Backspace/Delete로 내용만 삭제 (영역은 유지) — 이미 동작함
- **완전 삭제**: 도구상자 [지우기] 버튼 → 컨트롤 자체 제거
- Rust: `delete_header_footer_native` 함수 신규
- WASM: `deleteHeaderFooter` 바인딩
- 프론트엔드: `page:headerfooter-delete` 커맨드 + 도구상자 버튼

### 2. 이전/다음 머리말·꼬리말 이동

- 편집 모드에서 [이전]/[다음] 버튼으로 다른 머리말·꼬리말 간 이동
- 같은 구역 내에서 header→footer, footer→header 전환
- 다음 구역의 머리말/꼬리말로 이동
- 프론트엔드: `page:headerfooter-prev`, `page:headerfooter-next` 커맨드

### 3. 머리말/꼬리말 감추기 (현재 쪽만)

- 특정 페이지에서 머리말/꼬리말이 인쇄되지 않도록 감추는 기능
- 모델: `Section`에 `hide_header_pages: Vec<usize>` 등 플래그 추가
- 렌더러: 감추기 플래그가 있는 페이지에서 Header/Footer 노드 스킵
- UI: [쪽-감추기] 대화상자 또는 간소화 커맨드

### 4. 머리말/꼬리말 도구상자 고도화

현재 [머리말 | 닫기] 만 있는 도구상자를 한컴과 유사하게 확장:

```
[머리말(양쪽)] | [이전] [다음] | [닫기] [지우기]
```

## 우선순위

- 1순위: 삭제 + 도구상자 고도화 (핵심 편집 기능)
- 2순위: 이전/다음 이동 (편의 기능)
- 3순위: 감추기 (부가 기능)

## 제외 사항 (향후 타스크)

- 머리말/꼬리말마당 (템플릿) — 미리 정의된 양식 선택 기능
- 쪽번호 필드 삽입 — 별도 필드 시스템 필요
- 구역 설정의 "첫 쪽에만 넣지 않음" 옵션
- 눈금자 드래그로 머리말/꼬리말 영역 크기 조절
