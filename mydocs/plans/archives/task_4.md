# 타스크 4 - 수행 계획서: 렌더러 구현 (텍스트/표/폰트)

## 목표

파서(타스크 3)가 추출한 Document IR을 SVG로 정확하게 렌더링한다.
원본 HWP 문서의 텍스트, 표, 폰트가 SVG 출력에 올바르게 반영되어야 한다.

## 설계 원칙

HWP는 워드프로세서의 문서 구조체이다. 렌더링은 다음 순서를 반드시 지켜야 한다:

### 올바른 처리 순서

```
0단계: 스타일 목록 구성 (Style Resolution)
    DocInfo 참조 테이블 해소
    → FontFace[lang][id] → 폰트명
    → CharShape[id] → { 폰트명, 크기, 볼드, 이탤릭, 색상, 자간, 장평 }
    → ParaShape[id] → { 정렬, 줄간격, 들여쓰기, 여백 }
    → BorderFill[id] → { 테두리선, 배경색 }
    → 해소된 스타일 목록 (ResolvedStyleSet)

1단계: 문서 구조 구성 (Document Composition)
    Paragraph + ResolvedStyleSet
    → 줄별 텍스트 분할 (LineSeg.text_start 기반)
    → 줄 내 스타일 구간 분할 (CharShapeRef → ResolvedStyleSet 참조)
    → 인라인 컨트롤 위치 식별 (표/도형 삽입 위치)
    → 구성된 문서 구조 (ComposedDocument)

2단계: 렌더링 순서 계산 (Layout Composition)
    ComposedDocument 기반
    → 페이지네이션 (문단/표/도형의 페이지 배치)
    → 좌표 계산 (각 요소의 절대 위치)
    → 렌더 트리 구축

3단계: 출력 (SVG Rendering)
    렌더 트리 → 스타일 목록 참조 → SVG 요소 변환
```

### 현재 파이프라인의 문제

```
현재: Paragraph → Pagination → Layout (TextStyle::default) → SVG
         ↑ 스타일 목록 없음  ↑ 표 무시   ↑ 폰트/크기 무시

수정: DocInfo → 스타일 목록 → 문서 구조 구성 → Pagination → Layout (스타일 참조) → SVG
```

## 현황 분석

### 원본 문서 (통합재정통계 2014.8월) vs 현재 SVG 출력

| 항목 | 원본 | 현재 SVG | 상태 |
|------|------|----------|------|
| 제목 텍스트 ("통 합 재 정 통 계") | 있음 | 없음 | 미구현 |
| 부제목 ("2014. 8월(누계) 기준/잠정") | 있음 | 있음 | 정상 |
| 표 (통합재정수입/지출) | 있음 | 없음 | 미구현 |
| 본문 텍스트 (관리재정수지 설명) | 있음 | 있음 | 정상 |
| 각주 1)~5) | 있음 | 없음 | 미구현 |
| 폰트 (함초롬돋움 등) | 적용 | sans-serif 고정 | 미구현 |
| 텍스트 크기 (제목 24pt 등) | 다양 | 12px 고정 | 미구현 |
| 볼드/이탤릭 | 적용 | 미적용 | 미구현 |

### 근본 원인

1. **스타일 목록 미구성**: DocInfo 참조 테이블이 해소되지 않음 → 렌더링 시 스타일 룩업 불가
2. **문서 구조 미구성**: 줄별 텍스트 분할 및 스타일 구간 분할 없음 → 문단 전체가 하나의 TextRun
3. **인라인 컨트롤 미식별**: 문단 내 표/도형 삽입 위치가 파악되지 않음 → 표 렌더링 불가
4. **렌더링 파이프라인 단절**: DocInfo가 레이아웃/렌더링 단계에 전달되지 않음

## 구현 범위

### 1차 구현 (이번 타스크)

**스타일 목록 구성:**
- DocInfo → ResolvedStyleSet 변환
- CharShape + FontFace → 해소된 글자 스타일 (폰트명, 크기, 볼드/이탤릭, 색상, 자간)
- ParaShape → 해소된 문단 스타일 (정렬, 줄간격)
- BorderFill → 해소된 테두리/배경 스타일

**문서 구조 구성:**
- LineSeg.text_start 기반 줄별 텍스트 분할
- 줄 내 CharShapeRef 경계에 따른 다중 TextRun 생성 (각 TextRun이 스타일 ID 참조)
- 인라인 컨트롤 위치 식별 → 표/도형 삽입 위치 매핑

**렌더링 순서 계산:**
- 표 PageItem 생성 및 셀 레이아웃 계산
- 셀 내 문단 재귀 레이아웃
- 해소된 스타일을 렌더 트리에 반영

**SVG 출력:**
- 스타일 목록 참조하여 TextStyle 반영 (폰트, 크기, 볼드, 색상)
- 표 테두리 및 셀 텍스트 렌더링
- 텍스트 장식 (밑줄, 취소선)

### 미구현 (향후)

- 도형 렌더링 (직선, 사각형, 타원 등)
- 이미지 삽입 (Base64 data URI)
- 각주/미주 영역 렌더링
- 머리말/꼬리말 콘텐츠
- 텍스트 정렬 (가운데, 우측)

## 기대 결과

원본 HWP 문서의 다음 요소가 SVG에서 확인 가능해야 한다:
- 제목/본문 텍스트가 올바른 폰트, 크기, 볼드로 표시
- 표가 셀 테두리와 함께 렌더링
- 셀 내 텍스트가 표시
- 전체 페이지가 A4 크기로 정상 출력

## 검증 기준

- `rhwp export-svg` 출력 SVG에서 원본 스크린샷과 비교 가능한 수준의 텍스트/표 렌더링
- 기존 177개 테스트 유지 + 신규 테스트 추가
