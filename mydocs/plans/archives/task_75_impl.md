# 타스크 75: 표 앞 텍스트가 표 아래에 렌더링되는 문제 수정

## 배경

`samples/hwp-multi-001.hwp` 2페이지에서 표8(문단28) 앞에 위치한 텍스트 문단(문단22~27)이 표 아래에 렌더링되고, 첫 행은 표에 가려진다.

### 문서 구조

```
문단21: 표7 (4행×6열) — 1페이지
문단22~27: 텍스트만 포함 (※ 2024년 해외직접투자 상세통계는...)
문단28: 표8 (2행×6열) — 2페이지
문단29: 묶음 (그룹 이미지)
문단30: 표9 (1행×3열) — 3페이지
```

### 현재 렌더링 (잘못됨)

```
[표8] ← y≈109 (페이지 상단)
[텍스트 문단22~27] ← y≈568 (표 아래, 첫 행 가려짐)
[이미지 그룹]
```

### 기대 렌더링

```
[텍스트 문단22~27] ← 표 위에 위치
[표8]
[이미지 그룹]
```

### 문제 분석

| 위치 | 문제 |
|------|------|
| `pagination.rs:370-372` | `has_table` 문단은 초기 루프에서 건너뜀 — 그러나 텍스트 전용 문단(22~27)도 영향 받을 수 있음 |
| `pagination.rs:527-598` | 표가 포함된 문단 처리 시 Table 항목만 생성, 표 앞 텍스트(line 0) 미처리 |
| `layout.rs:260-274` | FullParagraph에 표가 있으면 spacing_before만 적용하고 skip |

---

## 구현 계획

### 1단계: 문제 재현 및 정확한 원인 분석

**목표**: 페이지네이션 디버깅으로 문단22~27이 페이지 아이템에 어떤 순서로 배치되는지 확인

- pagination.rs에서 페이지 2의 아이템 목록과 순서를 디버그 출력
- 문단22~27이 FullParagraph로 등록되는 시점과 표8(문단28)의 Table 아이템 순서 비교
- 정확한 원인 특정 (페이지네이션 순서 vs 레이아웃 y좌표 계산)

### 2단계: 페이지네이션/레이아웃 수정

**목표**: 표 앞 텍스트 문단이 올바른 순서로 렌더링되도록 수정

가능한 원인별 수정 방향:
- **페이지네이션 순서 문제**: 텍스트 문단이 Table 뒤에 배치되는 경우 → 아이템 삽입 순서 수정
- **레이아웃 y좌표 문제**: 아이템은 올바른 순서이지만 y 계산이 잘못된 경우 → para_start_y 계산 수정
- **표+텍스트 혼합 문단 문제**: 문단28에 텍스트와 표가 함께 있어 line 0 텍스트가 누락되는 경우 → PartialParagraph(line 0) 추가

### 3단계: 빌드 및 검증

1. `docker compose --env-file /dev/null run --rm test` — 전체 테스트 통과
2. `samples/hwp-multi-001.hwp` SVG 내보내기 → 2페이지에 텍스트가 표 위에 렌더링 확인
3. 기존 문서 SVG 내보내기 → 회귀 없음 확인
