# 타스크 40: HWP 저장 기초 기능 구현 - 수행계획서

## 배경

타스크 38에서 빈 HWP 문서에 텍스트를 삽입 후 저장한 파일이 한글 프로그램에서 열리지 않는 문제를 발견하고 수정했다. 이 경험에서 다음 문제를 확인했다:

- HWP 5.0 스펙 문서에 부정확하거나 모호한 표기가 있다
- 저장 파일의 한글 프로그램 호환성을 검증하는 체계적 방법이 없다
- 컨트롤별 저장 시 필수 레코드에 대한 가이드가 없다

## 목표

1. HWP 5.0 바이너리 스펙과 3.0 HWPML 스펙을 교차 검증하여 스펙 문서 보완
2. 빈 HWP 문서에 컨트롤을 하나씩 추가 저장하면서 한글 프로그램 오픈 검증
3. 각 컨트롤별 저장 기술 가이드 문서 작성

## 산출물

| 산출물 | 경로 |
|--------|------|
| 컨트롤별 저장 기술 가이드 | `mydocs/tech/hwp_save_guide.md` |
| 스펙 문서 수정 (교차 검증) | `mydocs/tech/hwp_spec_5.0.md` |
| 검증용 HWP 파일 | `output/save_test_*.hwp` |
| 코드 수정 (발견 버그) | `src/serializer/*.rs`, `src/wasm_api.rs` |

## 구현 단계 (5단계)

1. 스펙 교차 검증 및 기술 문서 초안
2. 기본 저장 검증 (텍스트만)
3. 표(Table) 컨트롤 저장 검증
4. 이미지(Picture) 컨트롤 저장 검증
5. 기타 컨트롤 저장 검증 및 최종 문서화

## 검증 방법

- 각 단계별 생성 HWP 파일 → 한글 프로그램 정상 오픈 확인
- `cargo test` 전체 테스트 통과
- WASM 빌드 성공
