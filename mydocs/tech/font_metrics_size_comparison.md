## 5. 경쟁 우위 분석

### rhwp 장점
- WASM 전체 크기가 폴라리스의 **1/10** (1.83 MB vs 19 MB)
- 한컴보다 **폰트 커버리지 13% 높음** (386 vs 342 패밀리)
- **단일 바이너리** 로드 (한컴은 387개 모듈 순차 로드 필요)
- 네이티브 빌드에서도 메트릭 사용 가능 (한컴은 브라우저 전용)

### 한글 메트릭 품질
- 음절분해 압축(cho×jung×jong 그룹): 최대 72 대표 폭으로 11,172 음절 커버
- 대부분 한글 폰트가 고정폭(uniform) → 오차 0
- 가변폭 폰트(Yj 시리즈 등): 최대 오차 8 em단위 (em=1000 기준 0.8%)

### 기본 폰트 우선 배치
배열 앞쪽에 HWP 기본/태블릿 폰트 배치로 `find_metric()` 선형 검색 최적화:
1. HCR Batang/Dotum (함초롬바탕/돋움) — HWP 기본
2. Malgun Gothic (맑은 고딕) — Windows 시스템 기본
3. Haansoft Batang/Dotum — 한소프트 레거시
4. NanumGothic/NanumMyeongjo — 인기 한글 폰트
5. **Noto Sans KR / Noto Serif KR** (R+B) — Android/Chrome OS/태블릿 기본
6. Arial, Times New Roman, Calibri, Verdana, Tahoma — 주요 영문 폰트
7. Batang, Dotum, Gulim, Gungsuh — 레거시 한글 폰트

### Noto CJK 추가 근거
- Android 태블릿/Chrome OS의 기본 한글 폰트
- Google Docs, Slides 등에서 기본 사용
- 오픈소스(OFL 라이선스)로 법적 부담 없음
- Regular + Bold 변형 커버 (23,174 글리프, 한글 11,172 전수 포함)
- 고정폭 한글 → 메트릭 데이터 극소 (폰트당 +3 KB WASM 증분)
