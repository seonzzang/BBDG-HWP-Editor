# RHWP Validation Modal Future Improvements

Date:

- `2026-04-24`

Project:

- `RHWP Integration Preservation Framework`
- `RHWP 엔진 통합 보존 프레임워크`

## 목적

현재 비표준 문서 경고 모달은 기능적으로는 정상 동작하지만,
일반 사용자 관점에서는 문구가 다소 기술적이고 딱딱하게 느껴질 수 있다.

이 문서는 해당 개선 사항을 즉시 작업하지 않고,
추후 여유가 있을 때 진행할 수 있는 UX 개선 과제로 보존한다.

## 범위

이 개선은 `RHWP 엔진 수정`이 아니라 `앱 UI/UX 레이어 개선`이다.

수정 대상 예상 파일:

- `rhwp-studio/src/ui/validation-modal.ts`
- 필요 시 `rhwp-studio/src/app/document-validation.ts`

건드리지 않는 대상:

- `rhwp` 코어 엔진
- validation warning 감지 로직 자체
- warning report 생성 규칙

즉, 엔진은 그대로 두고, 엔진이 반환한 경고를 사용자에게 더 자연스럽게 보여주는 방식만 개선한다.

## 현재 상태

현재 경고 모달은 아래와 같은 특징을 가진다.

- 제목이 기술적이다
  - 예: `HWPX 비표준 감지`
- 본문 설명이 명세/비표준 중심 표현이다
- `lineseg`, `textRun reflow` 같은 내부 용어가 노출된다
- 일반 사용자에게는 “오류”처럼 느껴질 수 있다

기능적으로는 정상이다.

- 새 문서에서는 뜨지 않아야 한다
- 실제 비표준 문서에서는 떠야 한다
- 자동 보정 / 그대로 보기 선택은 유지되어야 한다

## 개선 목표

### 1. 첫 화면 문구를 사용자 친화적으로 변경

기술 용어 대신 사용자 관점의 문장으로 바꾼다.

예시 방향:

- `HWPX 비표준 감지` -> `문서 보정 권장`
- `이 문서는 HWPX 명세를 일부 준수하지 않는 값을 포함합니다` ->
  `이 문서는 일부 서식 정보가 표준 형식과 다르게 저장되어 있어 화면 표시가 달라질 수 있습니다`

### 2. 내부 용어는 상세 보기로 내리기

처음부터 `lineseg`, `textRun reflow` 같은 용어를 전면에 노출하지 않는다.

권장 방식:

- 요약 영역: 사람말
  - `줄 배치 정보 보정 필요: 1건`
- 상세 보기: 기존 기술 표현 유지
  - `lineseg 가 문단당 1개 (한컴 textRun reflow 의존)`

### 3. 버튼 문구를 더 자연스럽게 정리

현재 버튼도 기능상 문제는 없지만,
조금 더 자연스러운 표현으로 바꿀 수 있다.

예시 후보:

- `자동 보정 (권장)` 유지 또는 `보정 후 열기`
- `그대로 보기` -> `그대로 열기`

### 4. 오류처럼 느껴지지 않게 톤 조정

이 모달은 “문서 로드 실패”가 아니라 “표시 품질 안내”에 가깝다.

따라서 다음 방향을 고려한다.

- 경고는 주되 과도하게 불안감을 주지 않기
- 자동 보정이 왜 필요한지 짧고 명확하게 설명하기
- 문서 열기 자체는 가능하다는 점을 분명히 하기

## 추천 문안 방향

현재 기준 추천 1안:

- 제목: `문서 보정 권장`
- 본문:
  - `이 문서는 일부 서식 정보가 표준 형식과 다르게 저장되어 있어 화면 표시가 달라질 수 있습니다.`
  - `보다 안정적인 표시를 위해 자동 보정을 권장합니다.`
- 요약:
  - `줄 배치 정보 보정 필요: N건`
- 버튼:
  - `자동 보정 (권장)`
  - `그대로 열기`
- 상세 보기:
  - 현재 기술 정보 유지

## 작업 시 주의사항

### 1. RHWP 엔진은 건드리지 말 것

이 개선은 엔진 경고 감지 로직을 바꾸는 작업이 아니다.

금지:

- warning 산출 규칙 변경
- 엔진 내부 validation 판단 기준 변경
- RHWP 코어 파일 수정

### 2. 정보 손실은 금지

일반 사용자에게는 쉬운 문구로 보여주더라도,
상세 보기에서는 기존 기술 정보를 유지해야 한다.

이유:

- 디버깅
- 개발 검증
- 사용자 제보 재현

### 3. 새 문서 예외는 유지

이미 수정된 정책:

- 새 문서에서는 비표준 경고 모달이 뜨지 않아야 한다
- 실제 비표준 문서에서는 떠야 한다

이 동작을 회귀시키면 안 된다.

## 우선순위

현재 우선순위는 낮다.

분류:

- `즉시 필수`: 아님
- `추후 UX 개선`: 맞음
- `엔진 통합 보존 프레임워크 원칙 위반`: 아님

즉, 기능 안정화와 핵심 검증이 우선이고,
이 문서는 나중에 여유가 생겼을 때 진행할 숙제로 본다.

## 완료 기준

추후 작업 시 완료로 보기 위한 기준:

- 새 문서에서는 경고 모달이 뜨지 않는다
- 실제 비표준 문서에서는 경고 모달이 뜬다
- 첫 화면 문구가 일반 사용자 친화적으로 개선된다
- 상세 보기에는 기술 정보가 유지된다
- 버튼 의미가 명확하다
- 기존 자동 보정 / 그대로 보기 기능은 유지된다
- RHWP 코어는 수정하지 않는다
