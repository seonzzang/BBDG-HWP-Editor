# HWP 표 렌더링 기술 지침서

## 개요

이 문서는 HWP 파일의 표(Table) 객체를 렌더링할 때 참고해야 할 기술 지침을 정리한다.
**HWP 공식 스펙 문서에는 오류 또는 잘못된 해설이 포함되어 있으므로**, 실제 HWP 바이너리 데이터와 한/글 프로그램의 동작을 기준으로 확인된 사항만 기록한다.

---

## 1. BorderFill 레코드 바이너리 구조

### 스펙 문서 오류

HWP 5.0 스펙 문서(표 25)는 4방향 테두리 데이터를 **순차 배열 형식**(`UINT8 array[4]` × 3)으로 설명하지만, 이는 **오류**이다.

### 실제 바이너리 레이아웃: 인터리브 형식

```
오프셋  크기   설명
──────  ─────  ──────────────────────────────
0       2      UINT16 속성 (attr)
2       6      좌측 테두리: 종류(1) + 굵기(1) + 색상(4)
8       6      우측 테두리: 종류(1) + 굵기(1) + 색상(4)
14      6      상단 테두리: 종류(1) + 굵기(1) + 색상(4)
20      6      하단 테두리: 종류(1) + 굵기(1) + 색상(4)
26      1      대각선 종류
27      1      대각선 굵기
28      4      대각선 색상
32      가변    채우기 정보
```

**각 테두리는 6바이트 단위로 반복** (종류 1 + 굵기 1 + 색상 COLORREF 4):

```rust
for i in 0..4 {
    let line_type = read_u8();   // 선 종류
    let width     = read_u8();   // 선 굵기 인덱스
    let color     = read_u32();  // COLORREF (LE)
}
```

### 검증 방법

인터리브 형식이 맞는지 확인하려면:
- 모든 테두리 색상이 정확히 `0x00000000`(검정)인지 확인
- 순차 배열 형식으로 읽으면 색상에 `0x00000001`, `0x00010100` 등 비정상 값이 나타남

## 2. 테두리선 종류 (BorderLineType)

### 스펙 문서 오류

스펙 문서는 인덱스 0부터 "실선(Solid)"으로 시작하는 것처럼 보이지만, 실제로는 **0 = 선 없음(None)**이다.

### 실제 매핑

```
인덱스  종류             한국어명
──────  ──────────────  ──────────
0       None            선 없음
1       Solid           실선
2       Dash            파선
3       Dot             점선
4       DashDot         일점 쇄선
5       DashDotDot      이점 쇄선
6       LongDash        긴 파선
7       Circle          원형 파선
8       Double          이중선
9       ThinThickDouble 가는선-굵은선 이중선
10      ThickThinDouble 굵은선-가는선 이중선
11      ThinThickThinTriple 가는선-굵은선-가는선 삼중선
12      Wave            물결선
13      DoubleWave      이중 물결선
14      Thick3D         3D
15      Thick3DReverse  3D 반전
16      Thin3D          3D 가는선
17      Thin3DReverse   3D 가는선 반전
```

### 주의사항

- `None(0)`일 때는 테두리를 렌더링하지 않아야 함
- 대부분의 셀 테두리는 `Solid(1)` + 굵기 `1`(0.12mm) 조합이 기본
- 한/글에서 표 속성 → 테두리 탭의 "종류" 목록 순서와 일치

## 3. 테두리선 굵기 매핑

HWP 스펙 표 28의 굵기 인덱스를 mm → px(96dpi) 변환:

```
인덱스  mm      px(@96dpi)
──────  ──────  ──────────
0       0.1     0.4
1       0.12    0.5
2       0.15    0.6
3       0.2     0.75
4       0.25    1.0
5       0.3     1.1
6       0.4     1.5
7       0.5     1.9
8       0.6     2.3
9       0.7     2.6
10      1.0     3.8
11      1.5     5.7
12      2.0     7.6
13      3.0     11.3
14      4.0     15.1
15      5.0     18.9
```

한/글 기본 테두리 굵기: **인덱스 1 (0.12mm = 0.5px)**

## 4. BorderFill ID 참조 규칙

- `border_fill_id`는 **1-indexed** (배열 접근 시 `-1` 필요)
- `border_fill_id == 0`은 BorderFill 참조 없음 → 테두리를 그리지 않음
- 표(Table)와 셀(Cell) 모두 자체 `border_fill_id`를 가짐
  - `table.border_fill_id`: **페이지 분할 시 경계선 용도** (시각적 외곽선이 아님!)
  - `cell.border_fill_id`: 개별 셀 테두리 스타일 (표의 시각적 테두리는 이것으로만 결정)

### 주의: table.border_fill_id는 시각적 외곽선이 아님

`table.border_fill_id`를 표 전체 외곽선으로 렌더링하면 안 된다. 이 값은 표가 페이지 경계에서 분할될 때 사용되는 경계선 스타일이다. 표의 시각적 외곽선은 **개별 셀의 `cell.border_fill_id`**로만 결정된다.

## 5. 표 객체 속성 구조 (한/글 대화상자 기준)

한/글에서 표 속성 대화상자의 각 탭별 주요 렌더링 속성:

### 5.1 기본 탭

| 속성 | 설명 | 렌더링 영향 |
|------|------|------------|
| 너비/높이 | 표 전체 크기 (mm) | 레이아웃 크기 계산 |
| 글자처럼 취급 | 인라인 vs. 플로팅 | 배치 방식 결정 |
| 본문과의 배치 | 텍스트 감싸기 방식 | 주변 텍스트 배치 |

### 5.2 여백/캡션 탭

| 속성 | 설명 | 렌더링 영향 |
|------|------|------------|
| 바깥 여백 | 표 외부 여백 (좌/우/상/하) | 표 주변 간격 |
| 캡션 위치/크기 | 캡션 배치 | 캡션 렌더링 위치 |

### 5.3 테두리 탭

| 속성 | 설명 | 렌더링 영향 |
|------|------|------------|
| 종류 | 선 종류 (0-17) | BorderLineType 결정 |
| 굵기 | 선 굵기 인덱스 (0-15) | stroke-width 결정 |
| 색 | 선 색상 (COLORREF) | stroke 색상 |
| 셀 간격 | 셀 사이 간격 (mm) | 셀 위치 계산에 반영 |

### 5.4 배경 탭

| 속성 | 설명 | 렌더링 영향 |
|------|------|------------|
| 채우기 종류 | 없음/색/그러데이션/그림 | 배경 렌더링 방식 |
| 면 색 | 배경 단색 | fill 색상 |

### 5.5 표 탭

| 속성 | 설명 | 렌더링 영향 |
|------|------|------------|
| 셀의 안 여백 | 셀 내부 텍스트 패딩 (mm) | 셀 내 텍스트 시작 위치 |
| 경계선 설정 | 페이지 분할 시 경계선 | 다중 페이지 표 |

## 6. 셀 레이아웃 계산

### 6.1 열 너비 계산

```
col_widths[col] = cell.width  (같은 열의 첫 번째 셀 기준, HWPUNIT → px)
col_x[0] = table_start_x
col_x[i+1] = col_x[i] + col_widths[i] + cell_spacing  (마지막 열 제외)
```

### 6.2 행 높이 계산

```
row_heights[row] = table.row_sizes[row]  (HWPUNIT → px)
row_y[0] = table_start_y
row_y[i+1] = row_y[i] + row_heights[i] + cell_spacing  (마지막 행 제외)
```

### 6.3 셀 간격 (cell_spacing)

- `table.cell_spacing` (HWPUNIT → px)
- 인접 셀 사이에만 적용 (외곽에는 적용하지 않음)
- 한/글 기본값: 0.00mm

### 6.4 셀 안 여백

- 한/글 기본값: 좌/우/상/하 각 0.50mm
- 셀 내 텍스트 시작 위치를 여백만큼 안쪽으로 이동
- `cell.padding_left`, `cell.padding_right`, `cell.padding_top`, `cell.padding_bottom`

## 7. SVG 렌더링 순서

표를 SVG로 렌더링하는 순서:

```
1. 표 전체 외곽선 (table.border_fill_id 참조, None이 아닌 경우만)
2. 각 셀 순회:
   a. 셀 배경 <rect> (fill_color 있는 경우)
   b. 셀 4방향 테두리 <line> (각 방향별, None 아닌 경우만)
   c. 셀 내 텍스트 <text>
```

### 7.1 테두리선 SVG 변환

| BorderLineType | SVG stroke-dasharray |
|---------------|---------------------|
| None          | (렌더링 안 함)      |
| Solid         | (없음 = 실선)       |
| Dash          | "6 3"               |
| Dot           | "2 2"               |
| DashDot       | "6 3 2 3"           |
| DashDotDot    | "6 3 2 3 2 3"       |
| Double 등     | Solid로 대체        |

## 8. 검증 체크리스트

표 렌더링 결과를 검증할 때 확인해야 할 사항:

- [ ] 모든 셀 테두리가 올바른 종류(실선/점선/없음)로 렌더링되는가
- [ ] 테두리 굵기가 원본과 일치하는가
- [ ] 테두리 색상이 정확한 검정(#000000)인가 (비정상 근사값이 아닌지)
- [ ] 셀 배경색이 올바르게 적용되는가
- [ ] 헤더 행의 아래쪽 굵은 테두리가 표시되는가
- [ ] 셀 내 텍스트 위치가 안 여백을 반영하는가
- [ ] 셀 간격이 올바르게 반영되는가
- [ ] 표 외곽선이 올바르게 표시되는가
